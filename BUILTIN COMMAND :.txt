BUILTIN COMMAND :
echo, cd, pwd, export, unset, emv, exit


echo with n option:
- echo display a line of text
- -n : do not output the trailing newline
echo print la string qui la suit.
	sans l'option -n, elle rajoute un \n a la fin.
	avec l'option -n, elle ne le rajoute pas.
	elle print les mots emtre " " et ceux qui ne le sont pas, sans imprimer les "":
		✗ echo -n "bonjour" ; echo "le monde"
			bonjourle monde
		✗ echo -n "bonjour" ; echo -n "le monde"
			bonjourle monde%
		✗ echo "bonjour" ; echo "le monde"
			bonjour
			le monde
		✗ echo "bonjour" echo "le monde"
			bonjour echo le monde
		✗ echo "bonjour" echo 'le monde'
			bonjour echo le monde
	si on rajoute une option qui n'existe pas ou une commande sans metacarac
	avant, elle la print comme n'importe quel mot peu importe sa position dans
	la phrase :
		✗ echo "bonjour" echo 'le monde'  rev
			bonjour echo le monde rev
		✗ echo "bonjour" echo 'le monde'  -rev
			bonjour echo le monde -rev
	meme si la commande existe, si elle n'est pas mise directement apres echo,
	elle n'est pas interpretee et est print avec le "-":
		✗ echo "bonjour" echo 'le monde'  -n
			bonjour echo le monde -n
		✗ echo - rev "bonjour" echo 'le monde'
			rev bonjour echo le monde
		✗ echo -rev "bonjour" echo 'le monde'
			-rev bonjour echo le monde
		✗ echo -rev - "bonjour" echo 'le monde'
			-rev - bonjour echo le monde
	si un " n'est pas ferme dans la phrase, elle ouvre un here_doc avec ecris
	"> " et recupere son contenu pour le print jusqu'a ce qu'on y ferme
	le "
	echo " rev - "bonjour" echo 'le monde'
		dquote> ehrugj
		dquote> ihogrenl
		dquote> "
 		rev - bonjour echo 'le monde'
		ehrugj
		ihogrenl
	en debut de string, rien n'est imprime jusqu'a avoir autre chose que des "'.
	Si les '" ont ete ouverts avant le carac, les guillemets qui ferment les 1er
	ne seront pas imprimes non plus
	✗ echo  ''''"  \  "''l'"-n bonjour ""echo "le monde'
